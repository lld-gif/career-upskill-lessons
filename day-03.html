DAY 03

AWS Cost Structures

Deep dive into AWS pricing models, reserved capacity, and cost optimization.

Learning Objectives

  • Navigate AWS's pricing dimensions across core services
  • Compare Reserved Instances, Savings Plans, and Spot pricing
  • Identify the top cost drivers in a typical AWS bill
  • Build a cost optimization strategy using AWS-native tools

AWS Pricing Dimensions

AWS pricing is multidimensional. Every service combines several billing axes, and understanding these is essential to controlling costs.

The Big Five Cost Centers

For most organizations, five services dominate the AWS bill:

  1. EC2 + EBS — Compute and attached storage (typically 40–50% of spend)
  2. RDS / Aurora — Managed databases (10–20%)
  3. S3 — Object storage (5–15%)
  4. Data Transfer — Egress and cross-region (5–15%)
  5. Lambda + API Gateway — Serverless compute (growing rapidly)

Commitment-Based Discounts

Reserved Instances (RIs)

Commit to a specific instance type in a specific region for 1 or 3 years:

Payment Option1-Year Discount3-Year Discount
No upfront~30%~40%
Partial upfront~38%~52%
All upfront~42%~60%

RIs are locked to instance family and region. Convertible RIs allow changing instance type but offer smaller discounts (~20–30%).

Savings Plans

Commit to a dollar-per-hour spend level rather than specific instances:

  • Compute Savings Plans: Apply across EC2, Lambda, and Fargate. Most flexible. Discounts of 30–50%.
  • EC2 Instance Savings Plans: Locked to instance family + region. Deeper discounts (up to 60%) but less flexible.
Decision framework: Start with Compute Savings Plans to cover your baseline. Layer EC2 Instance Savings Plans on top for workloads you're confident won't change. Use RIs only for RDS/ElastiCache where Savings Plans don't apply.

Spot Instances

Use spare AWS capacity at 60–90% discounts. AWS can reclaim instances with 2-minute notice.

  • Good for: batch processing, CI/CD, stateless web servers behind load balancers, data processing, ML training
  • Bad for: databases, stateful applications, anything that can't tolerate interruption
  • Strategy: diversify across instance types and AZs to reduce interruption risk

Cost Optimization Playbook

Quick Wins (Week 1–2)

  • Delete unused EBS volumes, unattached Elastic IPs, and idle load balancers
  • Stop or terminate non-production instances during off-hours
  • Review and right-size over-provisioned instances (AWS Compute Optimizer)
  • Enable S3 Intelligent-Tiering for unpredictable access patterns

Medium-Term (Month 1–3)

  • Purchase Savings Plans based on 30-day usage analysis
  • Implement auto-scaling for all production workloads
  • Migrate GP2 EBS volumes to GP3 (20% cheaper, better performance)
  • Review data transfer patterns — use VPC endpoints, CloudFront, same-AZ placement
  • Implement S3 lifecycle policies for infrequently accessed data

Structural (Month 3–12)

  • Adopt Graviton (ARM) instances — 20–40% better price-performance
  • Evaluate serverless for bursty workloads (Lambda, Fargate)
  • Consolidate accounts under AWS Organizations for volume discounts
  • Negotiate an Enterprise Discount Program (EDP) at $1M+ annual spend
  • Implement FinOps practices with cost allocation tags and team-level budgets

Enterprise Discount Programs

At scale ($1M+ annual spend), AWS offers negotiated agreements:

EDP (Enterprise Discount Program)

  • Commit to total annual spend in exchange for 5–15% discount on all services
  • Typically 1–3 year commitments
  • Discount applies on top of RIs and Savings Plans
  • Negotiate annually — leverage competitive pressure from Azure/GCP
Negotiation tip: AWS account teams are measured on committed spend growth. Time your negotiation around their fiscal year end (January) or when you have credible multi-cloud alternatives to demonstrate. Having Azure or GCP POCs running strengthens your position significantly.

  2. Explain why cross-AZ data transfer costs can be a hidden budget killer for microservices architectures. What's the mitigation?
  3. When should you choose an EC2 Instance Savings Plan over a Compute Savings Plan? Give a specific scenario.
  4. A startup is at $800K annual AWS spend and growing 100% YoY. Should they negotiate an EDP now? What commitment structure would you recommend?
  5. List three AWS cost optimization actions that require zero engineering effort and three that require architectural changes.
